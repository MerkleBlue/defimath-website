4:T1c39,<p>This is a pure optimization release. No new features, no API changes — same precision,
same behaviour, just fewer opcodes in <a href="/docs/math/stdnormcdf"><code>stdNormCDF</code></a>. Every option pricer
and greek built on it gets the savings automatically.</p>
<h2>Where the savings come from</h2>
<p>The previous <code>stdNormCDF</code> scaled its argument by <code>1/√2</code> and then called an
internal <code>erfPositiveHalf</code> helper that multiplied right back by <code>√2</code> — two unnecessary
MUL+DIV pairs in every call. Inlining the body into <code>stdNormCDF</code> and dropping the
helper (which had no other callers) eliminated the round-trip and lets each branch
write straight to the result. Net: <strong>74 gas per <code>stdNormCDF</code> call</strong>.</p>
<p>That single change propagates everywhere <code>stdNormCDF</code> is used inside the library.</p>
<h2>The numbers</h2>
<p>Biggest absolute wins are on the option pricers themselves — <code>callOptionPrice</code> and
<code>putOptionPrice</code> each call <code>stdNormCDF</code> twice (one for <code>Φ(d₁)</code>, one for <code>Φ(d₂)</code>):</p>

<p><code>gamma</code>, <code>vega</code>, <code>binaryDelta</code>, <code>binaryGamma</code>, <code>binaryVega</code> are unchanged — they're
built on the standard normal <em>PDF</em>, not the CDF, so this optimization doesn't reach
them.</p>
<p><code>impliedVolatility</code> shows the largest absolute drop because it calls <code>stdNormCDF</code>
inside every Newton-Raphson iteration — roughly 10 iterations to converge, each
saving ~74 gas.</p>
<h2>No behavioural change</h2>
<p>Same input domain, same return values to the last wei. The 396 tests across <code>Math</code>,
<code>Options</code>, and <code>Binary</code> all pass at the same precision thresholds. The strict
equality assertions in the perf suite caught the change automatically and forced
the threshold updates — that's exactly what they're for.</p>
